§ 02 / Pipelines
01
Live on Oracle Cloud Streaming Production

Real-Time Crypto Streaming Pipeline

Python · Redpanda · PostgreSQL · dbt · Airflow · Metabase · Docker · Oracle Cloud · GitHub Actions

A 9-service Docker Compose streaming pipeline ingesting live prices for ten crypto assets from CoinGecko every 60 seconds into Redpanda. The Python consumer implements poison-pill protection and retry logic for malformed JSON and invalid UTF-8 bytes, sustaining 14,400 updates/day without intervention.

Sub-five-minute end-to-end freshness from API to Metabase, delivered through a 3-layer dbt transform (staging → intermediate → mart) orchestrated by Airflow on a 5-minute schedule. A freshness gate auto-halts downstream runs on stale data. Deployed to an Oracle Cloud Always-Free ARM VM with Caddy + Let's Encrypt and three-layer security hardening (cloud firewall + iptables + Docker isolation). A 4-job GitHub Actions pipeline performs safety-gated SSH auto-deploys at ~90 s push-to-live latency, with 8 pytest unit tests passing in 0.14 s.

02
CDC Open source

Change-Data-Capture Pipeline

PostgreSQL · Debezium · Redpanda · Python · dbt · Airflow · Docker

Cut replication lag from hourly-batch (~60 min) to under 10 seconds by implementing log-based CDC: Debezium captures Postgres WAL events into Redpanda, then an idempotent Python consumer writes to the warehouse via primary-key upserts — eliminating duplicates on restart under at-least-once delivery semantics.

Built SCD Type 2 historical dimensions in dbt for point-in-time analytics on mutable source tables. Isolated malformed events through an Airflow dead-letter pattern, preventing bad records from ever blocking the pipeline.

Lakehouse Quality gates

Medallion Data Lakehouse with Quality Gates

MinIO · DuckDB · Pandas · Great Expectations · Airflow · Parquet

A Bronze → Silver → Gold lakehouse on MinIO with Parquet storage and DuckDB as the in-memory SQL engine, achieving cloud-native lakehouse semantics — object storage, columnar format, SQL compute — at zero infrastructure cost.

Data quality is enforced at the Silver gate via Great Expectations suites covering schema, nulls, referential integrity, and range rules, blocking bad partitions from ever promoting to Gold. Paired with partition-aware idempotent Airflow DAGs, the pipeline supports safe backfills and reruns without corruption.
